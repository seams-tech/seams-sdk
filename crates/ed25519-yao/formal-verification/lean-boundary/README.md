# Ed25519 Yao Rust-to-Lean Boundary

FV1 extracts exactly two implemented pure functions from the host-only clear
oracle:

- `ed25519_yao_generator::wrapping_add_le_256`;
- `ed25519_yao_generator::clamp_rfc8032`.

The checked-in Lean files are generated by the source revisions and scope in
[`toolchain.toml`](../toolchain.toml). Raw Charon LLBC remains transient because
its internal identifier order is nondeterministic. The package declares the
Aeneas Lean library explicitly and builds the named nonempty `Ed25519Yao` and
`Ed25519YaoBoundary` targets.

```sh
scripts/setup-aeneas.sh
cargo yao-fv aeneas-check
```

The check validates clean Git revisions, normalizes transient paths in LLBC,
regenerates the stable Lean artifacts, compares them byte for byte, confirms
both generated functions are referenced by the boundary, builds both targets,
and requires both top-level `.olean` files.

The extraction does not include SHA-512, scalar arithmetic, curve operations,
oracle outputs, circuit code, or protocol state. It provides translation
evidence for two helper implementations and carries no security claim.
