# Fixed-backend critical abort corpus v1

Status: executable local Phase D corpus. This index covers the critical
malformed-message, wrong-binding, reflection, replay/order, cryptographic
failure, and one-use terminal paths for the fixed Client and SigningWorker
construction. The complete source dispositions live in
[`upstream-check-inventory.md`](./upstream-check-inventory.md).

Every evidence name below is a Rust test or compile-fail fixture. The presign,
online, pool, and oracle suites must all pass together. A returned protocol
error consumes its role state; the persistence adapters then tombstone the
reserved or committed pair.

## Structurally unreachable source failures

Several upstream failures have no runtime mutation in the purpose-built API:

- participant count, duplicate participant, missing-self, threshold, and
  triple-threshold failures disappear with the two role-local constructors;
- wrong vector dimensions disappear after exact frame decoding into fixed
  arrays;
- aggregate `e = 0` with `eG = E` would require an identity `E`, which the
  validated triple boundary rejects;
- calling a later round, requesting output early, or reusing an owned state is
  rejected by Rust's type and move systems;
- reviving a tombstone has no API and is covered by a compile-fail fixture.

These are stronger fixed-parameter dispositions. The compile-fail/API suite is
their executable evidence.

## Claim boundary

The corpus proves controlled rejection for the selected mutations and state
transitions. It does not prove the cryptographic soundness of the corrected
OT/MTA composition, network availability, fairness, platform erasure, or
target-runtime constant-time behavior. Those remain the explicit assumptions
and review gates in [`assurance-ledger-v1.md`](./assurance-ledger-v1.md).
