# Encoding And Transcript Spec

This spec defines canonical context encoding, transcript binding, versioning,
and domain separation for Router/A/B split derivation.

## Encoding Rules

All V1 encoded fields use length-prefixed byte strings:

```text
lp(bytes) = u32be(len(bytes)) || bytes
```

Rules:

- `len` is the byte length.
- `u32be` is four bytes in big-endian order.
- Empty required fields are invalid before encoding.
- Unknown versions are invalid.
- Duplicate fields are invalid.
- Parsers reject trailing bytes.

## String Rules

All protocol strings are encoded as UTF-8 bytes after boundary validation.

Core derivation code must receive already-normalized strings. It does not
perform Unicode normalization. Boundary adapters must enforce the canonical
format for their network and identity provider.

Default limits:

| Field | Limit | Rule |
| --- | ---: | --- |
| `candidate_id` | 64 bytes | ASCII snake case |
| `request_kind` | 32 bytes | ASCII snake case |
| `correctness_level` | 64 bytes | ASCII snake case |
| `network_id` | 64 bytes | lowercase ASCII, digits, `_`, `-` |
| `account_id` | 256 bytes | canonical network account id |
| `account_public_key` | 512 bytes | canonical network public key string |
| `root_share_epoch` | 128 bytes | ASCII epoch label |
| `ceremony_id` | 128 bytes | ASCII id generated by Router |
| `deriver_set_id` | 128 bytes | ASCII deriver-set label |
| `deriver_index` | 16 bytes | decimal ASCII integer |
| `deriver_key_epoch` | 128 bytes | ASCII key epoch label |
| `quorum_policy` | 64 bytes | canonical policy string |
| role identity | 256 bytes | canonical role identity string |

Boundary adapters may enforce stricter limits.

## Role Identity Format

Role identities use this canonical string form:

```text
role:<role-name>:<deployment-id>:<key-fingerprint>
```

Examples:

```text
role:router:prod-us:sha256-abc123
role:deriver-a:prod-us:sha256-def456
role:deriver-b:prod-eu:sha256-987654
role:client:device-42:sha256-fedcba
role:signing-worker:prod-us:sha256-111111
```

Rules:

- `role-name` must match the role in the transcript.
- `deployment-id` is public metadata.
- `key-fingerprint` is a stable fingerprint of the envelope or signing key.
- Deriver A and Deriver B identities must differ.
- Role identity changes require a new transcript.

## Deriver Set V1

V1 is a strict A/B protocol release, but transcript internals are deriver-set
based to keep future N-of-N and t-of-N work possible.

V1 deriver-set rules:

- `quorum_policy` must be `all(2)`.
- exactly two deriver entries are present.
- deriver index `0` has role `deriver_a`.
- deriver index `1` has role `deriver_b`.
- Deriver A and Deriver B identities differ.
- each deriver entry binds its own key epoch.
- each deriver entry binds the hash of its role-specific encrypted envelope.

Future N-of-N can extend the deriver count under a new protocol version. t-of-N
requires a new protocol version, fresh vectors, quorum-selection review,
leakage analysis, replay and equivocation handling, and refresh/reshare
semantics.

## Context Encoding V1

Context V1 bytes:

```text
lp("router-ab-derivation/context/v1")
|| lp(candidate_id)
|| lp(request_kind)
|| lp(correctness_level)
|| lp(network_id)
|| lp(account_id)
|| lp(account_public_key)
|| lp(root_share_epoch)
|| lp(ceremony_id)
```

The current Rust helper `DerivationContext::encode_context_v1` must match this
layout.

## Context Digest V1

Context digest V1:

```text
SHA-256(
  lp("router-ab-derivation/context-digest/v1")
  || lp(context_v1_bytes)
)
```

Vectors include committed context digest values and the parser rejects digest
mismatches.

## Transcript Encoding V1

Transcript V1 bytes:

```text
lp("router-ab-derivation/transcript/v1")
|| lp(context_v1_bytes)
|| lp(router_id)
|| lp(deriver_set_id)
|| lp(quorum_policy)
|| lp(deriver_count)
|| for each deriver in ascending deriver_index:
     lp(deriver_index)
  || lp(deriver_role)
  || lp(deriver_identity)
  || lp(deriver_key_epoch)
|| lp(selected_signing_worker_id)
|| lp(selected_signing_worker_recipient_encryption_key)
|| lp(client_id)
|| lp(client_ephemeral_public_key)
```

The current Rust helper `transcript_binding_digest` must match this layout.

Public examples may still use `aEnvelope` and `bEnvelope`, but canonical
transcript bytes bind deriver-set id, deriver index, role, key epoch, quorum
policy, selected SigningWorker identity, selected SigningWorker recipient
encryption key, client identity, and client ephemeral public key.
Encrypted-envelope digests are assignment metadata outside the derivation
transcript.

## Transcript Digest V1

Transcript digest V1:

```text
SHA-256(transcript_v1_bytes)
```

Changing any field in `context_v1_bytes` or any role identity must change the
transcript digest under the hash collision-resistance assumption.

## Domain-Separation Registry

Reserved labels:

| Label | Purpose |
| --- | --- |
| `router-ab-derivation/context/v1` | Context encoding |
| `router-ab-derivation/context-digest/v1` | Context digest |
| `router-ab-derivation/transcript/v1` | Transcript digest |
| `router-ab-derivation/deriver-set/v1` | Deriver-set binding |
| `router-ab-derivation/envelope-aad/v1` | Envelope AAD |
| `router-ab-derivation/package-commitment/v1` | Delivery package commitment |
| `router-ab-derivation/minimum-level-c/v1` | Minimum Level C evidence |
| `router-ab-derivation/mpc-threshold-prf/v1` | Candidate A suite |
| `router-ab-derivation/split-root/v1` | Candidate B suite |

Candidate-specific derivation labels must be added to this registry before
implementation.

## Versioning

Versioned encodings are incompatible unless a spec explicitly defines a
translation rule. V1 parsers reject unknown context, transcript, envelope, and
evidence versions.

Compatibility code belongs at request or persistence boundaries. Core
derivation functions accept only current typed values.

## Required Vectors

Vectors must cover:

- context digest success
- transcript digest success
- changed candidate id
- changed request kind
- changed correctness level
- changed network id
- changed account id
- changed account public key
- changed root epoch
- changed ceremony id
- changed Router identity
- changed Deriver A identity
- changed Deriver B identity
- changed deriver-set id
- changed deriver index
- changed deriver key epoch
- changed quorum policy
- changed client identity
- changed client ephemeral public key
- changed selected SigningWorker identity
- changed selected SigningWorker recipient encryption key
- duplicate Deriver A/B identity rejection
- non-`all(2)` quorum rejection
- unknown version rejection
- empty required field rejection
